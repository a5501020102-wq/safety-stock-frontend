# 安全庫存計算系統 — 使用說明

> 版本：5.1.0

---

## 系統概述

本系統透過歷史銷貨資料，自動計算每個產品在每個倉庫應備的安全庫存量，協助採購人員做出補貨決策。

**三個步驟：**
1. 上傳銷貨 Excel
2. 設定計算參數
3. 查看結果並匯出

---

## 第一步：上傳檔案

### 銷貨資料（必要）

從 SAP 匯出的出貨明細，系統會自動辨識以下欄位：

| 欄位 | 可接受的欄位名稱 |
|------|----------------|
| 出貨點 | 出貨點、工廠、倉庫、倉別 |
| 料號 | 料號、品號、物料、物料編號 |
| 日期 | 出貨/退貨日期、出貨日期、交易日期、銷貨日期 |
| 數量 | 數量、出貨數量、銷貨數量 |
| 品名 | 品名、產品名稱（選填） |

- 檔案格式：`.xlsx` 或 `.xls`
- 大小限制：50MB
- 退貨（負數數量）會自動沖抵銷售，不需要事先處理

### 單價表（選填）

上傳後系統會用金額做 ABC 分類。不上傳則用數量分類。

### 庫存計畫（選填）

ERP 的供需計畫表，用於預測未來庫存走勢。

## 第二步：設定參數

### 計算模式

| 模式 | 說明 | 適用情境 |
|------|------|---------|
| **Compare（對比）** | 同時算分倉和總倉，顯示合併後能省多少庫存 | 評估是否值得合倉 |
| **Split（分倉）** | 每個出貨點獨立計算 | 日常補貨決策 |
| **Consolidated（總倉）** | 所有出貨點合併為一個倉 | 集中採購決策 |

### 計算粒度

| 粒度 | 說明 | 適用情境 |
|------|------|---------|
| **Monthly（月）** | 以月為單位統計需求 | 前置期 ≥ 30 天的產品 |
| **Weekly（周）** | 以 ISO 周為單位統計 | 前置期 7-30 天的產品 |
| **Daily（日）** | 以日為單位統計 | 前置期 < 7 天，快速進貨的產品 |

### 時間參數

| 參數 | 說明 | 範圍 |
|------|------|------|
| **Lead time（前置期）** | 下單到到貨的天數 | 1-365 天 |
| **Minimum periods（最少期數）** | 至少要有幾個期間有出貨才納入計算 | 0-12 |
| **Working days / month（每月工作天）** | 一個月有幾個工作天（僅月粒度顯示） | 1-31，預設 30 |

**Working days 的影響：**
- 設 30（預設）= 用日曆天計算
- 設 22 = 用工作天計算，日均需求會較高，安全庫存也會略高
- 此設定影響所有計算結果（SS、ROP、Max、Daily）

### ABC 服務水準

ABC 分類會根據產品的銷售金額（或數量）將產品分為三級，每級使用不同的安全係數：

| 分類 | 佔比 | 意義 | 預設 Z 值 | 服務水準 |
|------|------|------|----------|---------|
| **A 類** | 銷售額前 80% | 重點產品，不能缺貨 | 2.05 | 98% |
| **B 類** | 銷售額 80-95% | 一般產品 | 1.65 | 95% |
| **C 類** | 銷售額 95-100% | 次要產品，偶爾缺貨可接受 | 1.28 | 90% |

Z 值越高 → 安全庫存越多 → 缺貨機率越低。

### 統計過濾

| 開關 | 說明 |
|------|------|
| **MAD 離群值檢測** | 自動偵測並排除異常月份（例如某月突然出貨 10 倍），避免安全庫存被極端值拉高 |
| **移動平均** | 用滾動窗口平滑月與月之間的波動，適合需求不穩定但有趨勢的產品 |

### 日期範圍

- **From / To**：指定要用哪段期間的歷史資料計算
- 空白 = 使用上傳資料的完整範圍
- 格式：`2026/01/01`

### 季節性過濾（進階）

收合在「Advanced · Seasonal filter」底下。勾選的月份才會納入計算。

用途：某些產品有明顯季節性（例如冬天賣得好、夏天不動），只勾選旺季月份可以算出更貼近實際的安全庫存。

注意：這裡的「月份」是跨年的。勾選 1、2、3 月 = 使用**所有年份的** 1-3 月資料，不是只有今年。

### 趨勢偵測

| 選項 | 說明 |
|------|------|
| **None** | 不計算趨勢（預設） |
| **Short-term** | 比較資料的前半段和後半段，顯示需求變化百分比 |
| **YoY** | 比較最近兩年的同月份資料（需要 2 年以上的資料） |

趨勢結果顯示在表格的 TREND 欄：
- `+30%` = 需求上升 30%
- `-22%` = 需求下降 22%
- `New` = 新品項（之前沒有出貨，最近才開始）
- `Discontinued` = 停用品項（之前有出貨，最近完全沒有）
- `—` = 資料不足或不適用

### 分類前置期（進階）

收合在「Advanced · Category lead times」底下。如果系統載入了料號主檔，會顯示產品分類列表，每個分類可以設定不同的前置期。

例如：電線類 LT=45 天、PVC 管材 LT=14 天。

空白 = 使用全域的前置期設定。

| 欄位 | 說明 | 計算方式 |

## 計算公式

### 安全庫存 (SS)

```
SS = Z × 標準差 × √(前置期天數 / 每期天數)
```

**白話解釋：** 安全庫存 = 服務水準係數 × 需求波動程度 × 前置期調整因子

| 因素 | 影響 |
|------|------|
| Z 值越高 | SS 越大（備越多貨，越不容易缺） |
| 標準差越大 | SS 越大（需求越不穩定，要備越多） |
| 前置期越長 | SS 越大（等貨時間越長，風險越高） |

### 再訂購點 (ROP)

```
ROP = 日均需求 × 前置期天數 + 安全庫存
```

**白話解釋：** 庫存降到這個量時就該下單了。因為下單後還要等前置期那麼多天才到貨，這段期間的需求 + 安全緩衝 = ROP。

### 最大庫存 (Max)

```
Max = ROP + 前置期需求量
```

**白話解釋：** 補貨到達後的目標庫存上限。等於 ROP + 再一個前置期的需求量。

### 日均需求 (Daily)

```
Daily = 平均需求 / 每期天數

每期天數：
  月粒度 = Working days 設定值（預設 30）
  周粒度 = 7
  日粒度 = 1
```

### 變異係數 (CV)

```
CV = 標準差 / 平均需求
```

| CV 範圍 | 意義 |
|---------|------|
| < 0.3 | 需求穩定 |
| 0.3 - 0.5 | 中等波動 |
| > 0.5 | 需求波動大，安全庫存會較高 |

### 離群值檢測 (MAD)

```
1. 取出有出貨的期間（排除零需求月）
2. 計算中位數和 MAD（中位數絕對偏差）
3. 超過 中位數 ± 3倍 MAD 的值視為離群值
4. 離群值的位置填回 0，保持期間數不變
5. 用處理後的完整序列計算平均值和標準差
```

**白話解釋：** 如果某個月出貨量異常高（例如大單），系統會把那個月的數據排除，避免安全庫存被異常值拉高。零出貨的月份不會被當成異常。

### 趨勢偵測

**Short-term（短期趨勢）：**
```
將資料切成前半段和後半段
趨勢 = (後半段平均 - 前半段平均) / 前半段平均 × 100%
```

**YoY（年度同期比）：**
```
只比較最近兩年中「同月份」的資料
趨勢 = (今年平均 - 去年平均) / 去年平均 × 100%

規則：
- 只有兩年都有資料的月份才會被比較
- 資料超過兩年時，只取最近兩年
```

## 匯出功能

### Excel 匯出

點擊結果區的 **Excel** 按鈕，下載完整的計算結果。

對比模式會產生三張工作表：
1. **對比摘要** — 分倉 vs 總倉的庫存節省統計
2. **分倉計算** — 每個倉庫每個 SKU 的完整結果
3. **總倉計算** — 合併後的結果

欄位標題會根據計算粒度自動調整：
- 月粒度：活躍月數、月平均需求
- 周粒度：活躍周數、周平均需求
- 日粒度：活躍天數、日平均需求

### SAP MM17 匯出

點擊 **SAP MM17** 按鈕，產生可直接上傳到 SAP 的安全庫存設定檔。

格式：

| MATNR | WERKS | EISBE |
|-------|-------|-------|
| 料號 | 出貨點代碼 | 安全庫存量 |

**倉別選擇：** 使用匯出列旁邊的下拉選單選擇要匯出哪個倉的資料。選 "All sites" 會匯出所有倉。

## 常見問題

### 為什麼安全庫存是 0？

可能原因：
1. **只有 1 個期間的資料** — 算不出標準差。增加資料範圍或用更細的粒度。
2. **所有期間的出貨量完全相同** — 標準差 = 0，沒有波動就不需要安全庫存。
3. **日期範圍設太窄** — 檢查 Date Range 是否只涵蓋了 1 個月/周/天。

### 為什麼 ABC 全部顯示 A？

只有很少的 SKU 時，累計佔比很快就超過 80%，大部分會被歸為 A。這是正常的。上傳更多 SKU 的資料後，ABC 分布會更均勻。

### 為什麼 Trend 顯示 "—"？

可能原因：
1. 趨勢模式設為 None
2. 資料期間少於 4 個（前後半各不到 2 個，無法比較）
3. 月份篩選不連續（如只選 1、5、9 月），短期趨勢無法計算
4. YoY 模式但資料不到 2 年

### Compare 模式的節省量代表什麼？

分倉安全庫存的總和 vs 總倉安全庫存的差額。總倉合併後，因為各倉的需求波動會互相抵銷（風險池化效應），通常能減少 20-30% 的安全庫存。

這個數字幫助你評估「合倉」是否值得。
